# plan.md Acceptance Criteria

Use this reference when creating or reviewing `plan.md` in any seeded eval.

## Must-Have Content

- The plan states the role-specific objective clearly.
- The plan lists exact identifiers, labels, and file names used elsewhere in the handoff.
- The plan is consistent with the current seeded revision.
- The plan avoids placeholders and vague summaries where exact contract data is available.

## Acceptance Criteria

- Headings match the active role contract.
- Every required inventory label or part identifier appears exactly where the contract expects it.
- The plan matches the YAML files, scripts, and review artifacts.
- The plan does not invent parts, zones, motion, or pricing.

## Reviewer Use

- Reject plans that are incomplete, inconsistent, or drift from the seeded workspace.
- Reject plans that use generic prose where the contract expects exact names or counts.
