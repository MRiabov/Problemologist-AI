# todo.md Acceptance Criteria

Use this reference when creating or reviewing `todo.md` in any seeded eval.

## Must-Have Content

- The TODO list reflects the current stage only.
- The TODO items are ordered in execution order.
- The TODOs are concrete and actionable.
- The file does not keep stale bookkeeping from prior stages.

## Acceptance Criteria

- The file uses checkboxes consistently.
- Completed items are marked consistently with the current revision state.
- The list matches the plan and the seeded artifacts.
- The TODO list does not introduce unsupported work outside the current contract.
